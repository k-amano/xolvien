## 優先度: 高（動作品質・致命的な問題）

### ~~H1: 文字化けの修正~~ ✅ 対応済み（2026-04-19）

- `docker_service.py` の `execute_command_stream()` で `codecs.getincrementaldecoder` を使用するよう修正
- チャンク境界でのマルチバイト文字分割を正しく処理

### H2: テスト結果サマリーの表示

- **現象**: テスト完了メッセージだけが出て、passed / failed 件数などの集計が画面に表示されない
- **対応**: 実装確認パネル（`reviewing` 状態）にテスト結果サマリーを表示する

### H3: テストケース修正UIの改善

- **現象**: 「修正を依頼」ボタンはあるが、修正内容の入力に `window.prompt` を使用しており使いにくい
- **対応**: テストケース確認パネルにインライン入力欄（テキストエリア + 送信ボタン）を追加し、`window.prompt` を廃止する

## 優先度: 中（機能上の問題）

### M1: テスト種別の表示

- **現象**: ステータスバッジやバナーが「テスト」とだけ表示され、単体 / 結合 / E2E のどれかが分からない
- **対応**: `running_tests` ステートに `test_type`（unit / integration / e2e）を持たせ、バナー・バッジに反映する

### M2: 処理中の進捗表示

- **現象**: 処理中に動いているか止まっているか分からない
- **対応**: 各フェーズの開始時にバナーテキストを更新する（例：「テストコード生成中」「テスト実行中」「自動修正中 1/3」）

### M3: テストケースの表形式出力

- **現象**: 現在はMarkdownのチェックリスト形式で出力される
- **対応**: `generate_test_cases()` のプロンプトを修正し、以下の列を持つMarkdown表形式で出力させる
  - 対象画面 / テスト項目 / 操作方法 / 期待動作

### M4: テスト結果の集計表

- **現象**: ログにテキストで流れるだけで集計表がない
- **対応**: 実装確認パネルおよびテストレポート（Markdown ファイル）にテストケースごとの結果・判定・実行日時を含む集計表を追加する

---

## 優先度: 低（UX改善）

### L1: 右ペインのチャット型レイアウト刷新

- **現象**: メッセージ入力欄が狭い。処理フェーズのタイミングによって入力欄が消えることがある
- **対応**: `instruction-panel` のレイアウトを全面的にチャット型に刷新する（会話履歴が上から時系列で積まれてスクロール、入力欄は常に下部固定）

## フェーズ2: 結合テスト

単体テストと同様の流れで、DB・API 接続を伴うテストを実装する。

**バックエンド:**
- `claude_service.py` に `run_integration_tests()` を追加
  - `TestType.INTEGRATION` で `TestRun` を作成
  - テストコンテナ内で DB や API を起動した状態でテスト実行

- `instructions.py` に `POST /run-integration-tests` エンドポイントを追加

**フロントエンド:**
- ステップバーの「結合テスト」ステップをアクティブ化
- 単体テスト完了後に結合テストへ進むフローを追加
- `PromptState` に `'running_integration_tests'` を追加

---

## フェーズ3: E2E テスト（Playwright）

**バックエンド:**
- `claude_service.py` に `run_e2e_tests()` を追加
  - `TestType.E2E` で `TestRun` を作成
  - Playwright のスクリーンショットを自動修正フィードバックに含める
  - テストレポートにスクリーンショットのパスを記録

**フロントエンド:**
- ステップバーの「E2Eテスト」ステップをアクティブ化
- E2E テスト結果にスクリーンショットプレビューを表示

---

## PR 自動作成

テスト完了・ユーザー承認後に GitHub PR を自動作成する。

**バックエンド:**
- `git/push` エンドポイントを拡張、または新規に `POST /git/create-pr` エンドポイントを追加
- コンテナ内で `gh pr create` を実行
- PR タイトル・本文を Claude が生成

**フロントエンド:**
- 実装確認画面の「承認」後に PR 作成オプションを表示

---

## GitHub Issue 連携

GitHub Webhook でイシューを受け取り、タスクを自動生成・実行する。

**バックエンド:**
- `POST /api/v1/webhooks/github` エンドポイントを追加
- Issue の本文をタスクの指示として使用して自動フローを開始
